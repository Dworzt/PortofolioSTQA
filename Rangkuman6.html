Kelompok 6: API Testing

Panduan lengkap untuk testing API: definisi tujuan, tipe pengujian (functional, contract, performance, security), strategi otomatisasi, contoh skenario, dan praktik untuk menjaga API robust dan terpercaya.

Mengapa API Testing Penting?

API adalah lapisan yang mengikat frontend, backend, dan layanan pihak ketiga. Error di API sering menyebabkan banyak kegagalan fungsi. API tests cepat, deterministic, dan bagus untuk contract & integration checks.

Tipe Pengujian API

  • Functional tests: verifikasi endpoint berperilaku sesuai spesifikasi.
  • Contract tests: memastikan konsumen/penyedia API setuju terhadap schema & contract (mis. Pact).
  • Performance tests: beban / stress pada endpoint kritis (latency, throughput).
  • Security tests: authentication, authorization, input validation, injection tests.

Anatomi Request & Response (Detail)

Pastikan setiap tes memeriksa: method, endpoint, path params, query params, headers (content-type, auth), body, dan status code. Verifikasi juga schema (required fields, types) dan nilai penting.

  • Response time assertions (SLA): mis. 95th percentile < 500ms untuk GET /items
  • Schema validation: gunakan JSON Schema atau tools yang mendukung contract tests.

Contoh Skenario Pengujian

  1. Smoke: health-check endpoint → 200 OK
  2. CRUD flow: create → read → update → delete, verifikasi persistence
  3. Negative tests: invalid payload, missing auth, invalid param types
  4. Rate limit: melebihi batas dan periksa 429

Tools & Automasi

  • Exploratory: Postman (collections), Insomnia
  • Automasi: Newman (Postman CLI), REST Assured, Supertest + Jest, pytest + requests
  • Contract: Pact untuk consumer-driven contract testing
  • Performance: k6, JMeter

Security & Edge Cases

Uji autentikasi (expired tokens), authorization (akses ke resources terlarang), input validation (SQL/NoSQL injection), dan payload boundary cases (very large body, empty fields).

Best Practices

  • Isolasi environment: gunakan test/staging env, mock third-party yang tidak stabil.
  • Automate smoke tests on deploy; run full API test suite in CI nightly or per PR.
  • Use contract tests to avoid integration breakages between teams.
