Kelompok 5: Pengantar Unit Testing

Catatan mendalam tentang unit testing: tujuan, strategi penulisan tes, isolasi dependensi, contoh praktis, dan tips menjaga suite tes cepat dan andal.

Tujuan Unit Testing

  • Verifikasi perilaku unit terkecil (fungsi/metode) secara cepat.
  • Deteksi regresi sedini mungkin.
  • Mendokumentasikan kontrak fungsi (input/output yang diharapkan).

Prinsip Penulisan Tes yang Baik

  • Setiap tes hanya menguji satu skenario/perilaku.
  • Tes harus deterministik dan independen (dapat dijalankan berurutan atau paralel).
  • Gunakan nama tes yang jelas dan deskriptif.
  • Jaga agar fixture/setup singkat sehingga tes tetap cepat.

Pola: Arrange, Act, Assert (AAA) — Contoh

// Arrange
const input = {height:170, weight:65};
// Act
const result = calculateBMI(input);
// Assert
expect(result.bmi).toBeCloseTo(22.49, 2);

Contoh di atas memperlihatkan format AAA pada pseudocode test.

Isolasi, Mocking & Test Doubles

Untuk menjaga unit tests cepat dan deterministik, isolasi dependensi seperti database, network, atau waktu dengan mocks/stubs/fakes.

  • Mock: menggantikan dependensi dengan implementasi yang dapat dikontrol.
  • Stub: memberikan nilai yang sudah ditentukan.
  • Fake: implementasi ringan yang berfungsi seperti asli (mis. in-memory DB).

Strategi Suite & CI

Jaga agar unit tests cepat (<1s per test idealnya), jalankan di CI pada setiap commit. Pisahkan suite: unit (fast), integration (slower), E2E (slowest).

Framework & Tools — rekomendasi singkat

  • Java: JUnit + Mockito untuk mocking
  • JavaScript/Node: Jest (inkl. mocking), sinon untuk mocking khusus
  • Python: pytest + pytest-mock
