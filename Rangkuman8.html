Kelompok 8: Automation Testing (Cypress)

Catatan komprehensif Cypress: filosofi yang membedakan dari Selenium, alur eksekusi di dalam browser, fitur unik (time travel, automatic waits), dan praktik menulis tes E2E yang cepat dan tahan flake.

Kenapa Memilih Cypress?

Cypress berjalan di dalam browser sehingga memberikan kontrol dan observability yang lebih besar terhadap runtime dibanding tes berbasis remote WebDriver. Automatic waits dan test runner interaktif membantu debugging.

Arsitektur & Cara Kerja

Cypress berjalan di thread yang sama dengan aplikasi sehingga dapat memanipulasi network, clock, dan DOM secara real-time. Ini memungkinkan snapshot/time-travel debug dan mengurangi kebutuhan explicit waiting.

Fitur Utama & Praktis

  • Time travel: snapshot setiap langkah untuk debugging
  • Automatic waits: Cypress menunggu command & assertion terpenuhi
  • Network control: intercept, stub, dan mock API responses
  • Custom commands: abstraction layer untuk reuse

Menulis Tes yang Baik di Cypress

  • Gunakan data attributes untuk selector: cy.get('[data-cy=login-button]')
  • Manfaatkan cy.intercept() untuk mengemulasi responses dan menstabilkan pengujian
  • Pisahkan test utilities & custom commands untuk mempersingkat tes
  • Jangan mengandalkan state dari tes lain; setup state dalam beforeEach jika perlu

Strategi Testing & CI

Gunakan kombinasi unit, component, and E2E tests. Jalankan E2E minimal di PR (smoke) dan full suite pada pipeline nightly. Cypress Dashboard atau parallelization membantu menurunkan waktu run.

Debugging Tips

  • Gunakan time-travel snapshots di test runner
  • Gunakan cy.pause() dan cy.debug() untuk interaktif debugging
  • Periksa network logs dan console output yang disediakan oleh runner
